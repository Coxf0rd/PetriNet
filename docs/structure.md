# Архитектура проекта PetriNet

Проект построен вокруг двух основных подсистем: **модели сети Петри** (логика, расчёты, единицы данных) и **GUI** (`eframe`/`egui`), которая позволяет строить сети, запускать симуляцию и анализировать цепь Маркова.

## Структура каталогов

- `src/` — исходники, сгруппированные по функциональным блокам:
  - `src/model.rs` — ядро Petri Net (места, переходы, таблицы `pre`, `post`, `inhibitor`, `mo`). На основе этих данных строятся состояния и применяются переходы.
  - `src/markov.rs` — построение цепи Маркова: обход состояний, генерация матрицы `Q`, решение линейной системы для стационарного распределения.
  - `src/ui/` — графический интерфейс:
    - `app.rs` — точка входа, содержит перечень окон/режимов, запускает `PetriApp` и следит за состоянием приложения.
    - `app/graph_view.rs` — отрисовка графа дуг/узлов, включая марковские аннотации и связи.
    - `app/petri_app/` — основной модуль приложения, разбит на подпакеты:
      - `drawing/` — вся визуальная логика: `draw_menu.rs` (меню приложения), `draw_markov_window.rs` и `draw_place_props_window.rs` (оконные панели для марковской модели и свойств мест), `draw_sim_dialog.rs` (диалог симуляции) и т. д.
      - `helpers/` — утилиты `markov_placement_text.rs`, `format_marking.rs`, `node_color_text.rs` и т. п., которые помогают форматировать текст, цвета и подсказки.
      - `markov/` — вспомогательные функции для постановки марковских меток, расчёта ожидаемого числа токенов и распределений.
      - `netstar/`, `selection/`, `geometry/`, `clipboard/`, `file_ops/` — вспомогательные модули для экспорта, выбора, геометрии, буфера обмена и операций с файлами.

## Документация и планы

- `docs/markov_validation_plan.md`, `docs/task_plan_2026-03-08.md` — содержат цели и шаги по улучшению марковского анализа и прочих компонент.
- `docs/work.md` — оперативные заметки и план работ по проекту.
- `docs/structure.md` — этот файл как централизованный обзор архитектуры.

## Анализ марковской модели

- Калькуляция `MarkovChain` происходит в `src/markov.rs`. Там же реализован обход состояний и решение стационарного распределения методом Гаусса.
- В `src/model.rs` хранятся таблицы `pre`/`post`, `inhibitor`, а также обусловленные ограничения `mo`, которые влияют на возможность применения переходов.

## Интерфейс и визуализация

- `src/ui/app/petri_app/drawing/draw_markov_window.rs` — окно «Марковская модель», где отображаются распределение, граф состояний и выделения марковских меток. Именно здесь реализованы ScrollArea, Grid и вызов соответствующих хелперов.
- `draw_place_props_window.rs` дополняется чекбоксом для марковской метки и параметрами (положение), что влияет на визуальную аннотацию в окне.
- `draw_menu.rs` добавляет пункт меню/переключатель, открывающий окно анализа и запускающий расчёт.
- `helpers/markov_placement_text.rs` и `markov/helpers.rs` содержат логику подключения аннотаций к местам и текстов пояснения.

## Сборка и релизы

- Сборка осуществляется через `cargo` (check/test/format), а готовый portable-исполняемый файл создаётся через `build_portable_exe.ps1`.
- Текущая версия и бинарный файл обновляются при расширенных фиксим (см. версию в `Cargo.toml`/`Cargo.lock`).

## Совместная работа

- Все изменения, затрагивающие GUI, логические модели и скрипты, отражаются в `main` ветке. По инструкции из `AGENTS.md` при внесении изменений обязательно повышать версию, перезапускать сборку и тесты.

В этом файле — краткая карта, по которой можно ориентироваться, когда требуется добавить новое окно, расширить расчёты Маркова или отладить сигнал GUI ↔ модель.
