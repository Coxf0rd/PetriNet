# План: защита от дурака и поддержка марковских моделей

## Цель
1. Добавить «защиту от дурака» при вводе параметров (вкладки симуляции, свойства мест/переходов и др.), чтобы нематерильные значения автоматически скалировались к допустимому диапазону и подсвечивались при ошибке.
2. Построить граф состояний для сети Петри и решить уравнения Кольмогорова (переднее/обратное) на этом графе, чтобы выводить стационарное распределение марковской модели и предельные вероятности. Kolmogorov equations описывают эволюцию дискретных цепей Маркова на конечном пространстве состояний и позволяют получить стационарные вероятности через решение линейной системы [Wikipedia: Kolmogorov equations].

## Шаги
1. Проанализировать текущие поля ввода в `draw_sim_dialog`, `draw_place_props_window`, `draw_transition_props_window` и других UI-функциях: определить подозрительные диапазоны (`time_limit_sec`, `pass_limit`, `place capacity`, `delay`, параметры распределений и т.п.).
2. Написать утилен `clamp_input` (для `f64`, `u64`, `usize`) с защитой от NaN/Inf и сигналом об ошибке; добавить поддерживающие потомков описанные предупреждения, класс индикации неверного ввода (например, локальный текст/цвет).
3. Внедрить защиту во все формы: после `DragValue`/`TextEdit` скалировать и, при необходимости, активировать флаг `invalid` для красной метки. Проверить, что повторный ввод без чисел не ломает приложение.
4. Реализовать генератор графа Маркова:
   - Переписать или адаптировать логику `enabled_transitions`, чтобы принимать произвольный вектор маркеров и возвращать список достижимых переходов.
   - Провести обход в ширину (с ограничением на число состояний) и собирать ориентированный граф `StateId -> Vec<(StateId, rate)>`.
5. Составить и решить уравнение Кольмогорова для стационарного распределения: построить генераторную матрицу `Q`, дополнить уравнение `??=1`, решить методом Гаусса (невырожденная линейная система). Писать устойчиво для небольших матриц (напр., до 200 состояний).
6. Добавить новый UI-интерфейс (окно/вкладку), где можно запускать Markov-анализ (кнопка в меню «Результаты» или отдельный модуль), показывать список состояний и вероятностей, граф переходов, предупреждение, если пространство превышает лимит.
7. Написать тесты (`tests/markov.rs`) для генератора состояний и решения системы (например, простая сеть с фиксированным графом) и проверить, что стационарное распределение сходится.
8. Обновить README/документацию, повысить версию (0.7.13 > 0.7.14), собрать `PetriNet-0.7.14.exe` через `build_exe.ps1`, прогнать `cargo check`, `cargo test`, `rustfmt --check`, `rg -n ...`.
